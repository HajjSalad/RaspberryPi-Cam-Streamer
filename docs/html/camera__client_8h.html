Detailed Description

Public API for V4L2 camera streaming and LED control.

This header defines the interface for controlling a V4L2 camera device and a custom LED control driver.

◆ capture_frames()

int capture_frames ( struct camera_ctx *  ctx)

Captures and outputs video frames.

This function runs the main capture loop. It repeteadly:

  1. Dequeues a filled buffer using VIDIOC_DQBUF
  2. Writes the frame data to output stream (ex. stdout)
  3. Re-queues the buffer with VIDIOC_QBUF for reuse
Parameters
ctxPointer to the camera context structure that holds all session state.
Note
STREAMON must have been called before entering this loop.
Buffers must already be requested, mapped, and queued.

◆ map_buffers()

int map_buffers ( struct camera_ctx *  ctx)

Maps kernel-allocated V4L2 buffers into user space.

After buffers are requested with VIDIOC_REQBUFS (see request_buffers()), this function retrieves metadata for each buffer using VIDIOC_QUERYBUF and maps them into user space via mmap(). The mapped buffers allow the application to directly access video frames written by the camera driver without copies.

Parameters
ctxPointer to the camera context structure that holds all session state.
Returns
int
  • 0 on success
  • -errno on failure
Note
On success, the caller is responsible for unmapping the buffers using munmap() and freeing the buffers array.

Queues memory-mapped buffers to the video device.

After buffers are mapped into user space using map_buffers(), they must be queued to the kernel driver before streaming. This allows the driver to know which buffers are available for the camera to write captured frames into.
