int camera_init ( struct camera_ctx *  cctx)

Initialize and prepare the camera device for streaming.

Function Prototypes

This function sets up the camera context and performs the required sequence of V4L2 initialization steps.

On failure, the function automatically cleans up all resources allocated up to the failure point by calling close_camera().

Parameters
cctxPointer to the camera context structure that holds all the states.
Returns
0 on success Negative value on failure

int capture_frames ( struct camera_ctx *  cctx,
struct stream_ctx *  sctx,
struct pipeline_ctx *  pipeline 
)

Captures and outputs video frames.

This function runs the main capture loop. It repeteadly:

  1. Dequeues a filled buffer using VIDIOC_DQBUF
  2. Sends the YUYV frame for processing
  3. Re-queues the buffer with VIDIOC_QBUF for reuse
Parameters
cctxPointer to the camera context structure that holds camera sessions.
sctxPointer to the stream context containing stream session info.
pipelinePointer to the pipeline context containing thread and synchronization primitives
Note
STREAMON must have been called before entering this loop.
Buffers must already be requested, mapped, and queued.
Returns
0 on success, negative value on error.

◆ close_camera()

void close_camera ( struct camera_ctx *  cctx)

Stop the camera stream and release all associated resources.

This function performs full shutdown of the camera context:

  1. Stops the V4L2 video stream if it is running
  2. Unmaps and frees all MMAP buffers
  3. Closes the camera and control device file descriptors
Parameters
cctxPointer to the camera context structure that holds all the states.
Returns
